# Google Stitch MCP — Design Patterns & Features

Reference for UI patterns generated by Google Stitch (accessed via MCP protocol).
Stitch generates production-ready Tailwind CSS HTML from text prompts.

---

## Setup & Access

- **MCP Package**: `@_davideast/stitch-mcp`
- **Transport**: HTTP to `https://stitch.googleapis.com/mcp`
- **Auth**: API Key (preferred) or OAuth
- **Add to Claude Code**: `claude mcp add stitch --transport http https://stitch.googleapis.com/mcp -H "x-goog-api-key: YOUR_KEY"`
- **Tools**: `create_project`, `list_projects`, `generate_screen_from_text`, `list_screens`, `get_screen`, `edit_screens`, `generate_variants`

## Tech Stack

- **CSS Framework**: Tailwind CSS via CDN (`https://cdn.tailwindcss.com?plugins=forms,container-queries`)
- **Icons**: Google Material Symbols Outlined (`font-variation-settings: 'FILL' 1, 'wght' 400, 'GRAD' 0, 'opsz' 24`)
- **Fonts**: Inter (display), JetBrains Mono (mono), Playfair Display (serif quotes)
- **Dark Mode**: `class` strategy on `<html>` element
- **Config**: Inline `<script id="tailwind-config">` with custom design tokens

## UI Patterns by Section

### 1. Hero — 2-Column with Metric Cards

**Layout**: `grid lg:grid-cols-2 gap-16 items-center`
**Left column**: Badge pill + large title + subtitle + CTAs
**Right column**: 2x2 metric card grid

**Badge pill**:
```html
<div class="inline-flex items-center gap-2 border border-secondary/30 bg-secondary/10 px-4 py-1.5 rounded-full w-fit">
  <span class="w-2 h-2 rounded-full bg-secondary"></span>
  <span class="font-mono text-secondary text-xs uppercase tracking-wider font-bold">Label Text</span>
</div>
```

**Metric card (standard)**:
```html
<div class="bg-surface-dark border border-white/5 rounded-2xl p-6 flex flex-col justify-between hover:border-white/10 transition-colors">
  <span class="material-symbols-outlined text-white/40">icon_name</span>
  <div class="mt-8 font-mono">
    <div class="text-3xl font-bold text-white">8.4</div>
    <div class="text-primary text-xs uppercase tracking-widest mt-1">KM</div>
  </div>
</div>
```

**Metric card with circular progress ring (BPM)**:
```html
<div class="relative flex items-center justify-center w-24 h-24">
  <svg class="absolute inset-0 w-full h-full text-primary/20" viewBox="0 0 100 100">
    <circle cx="50" cy="50" r="45" fill="none" stroke="currentColor" stroke-width="4"/>
  </svg>
  <svg class="absolute inset-0 w-full h-full text-primary" viewBox="0 0 100 100">
    <circle cx="50" cy="50" r="45" fill="none" stroke="currentColor"
      stroke-dasharray="283" stroke-dashoffset="60" stroke-linecap="round" stroke-width="4"/>
  </svg>
  <span class="material-symbols-outlined text-primary text-3xl">favorite</span>
</div>
```

### 2. Features Grid — Material Icons + Stat Pill Badges

**Layout**: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6`

**Card structure**:
```html
<div class="flex flex-col rounded-2xl border border-card-border bg-card-bg backdrop-blur-sm p-8
            transition-all duration-300 hover:border-primary/30 group relative overflow-hidden">
  <!-- Hover gradient overlay -->
  <div class="absolute inset-0 bg-gradient-to-br from-primary/5 to-transparent
              opacity-0 group-hover:opacity-100 transition-opacity duration-500 pointer-events-none"></div>
  <!-- Icon -->
  <div class="text-primary mb-6">
    <span class="material-symbols-outlined text-4xl">monitor_heart</span>
  </div>
  <!-- Content -->
  <h2 class="text-white text-xl font-bold">Title</h2>
  <p class="text-slate-400 text-sm leading-relaxed">Description text.</p>
  <!-- Stat pill at bottom -->
  <div class="pt-6 border-t border-card-border flex items-center justify-between mt-auto">
    <span class="text-xs font-mono text-slate-400 uppercase tracking-wider">Label</span>
    <span class="text-sm font-bold text-primary bg-primary/10 px-3 py-1 rounded-full">99.6%</span>
  </div>
</div>
```

**Common Material icons used**:
- `monitor_heart` — Heart rate
- `schedule` — Sleep/time
- `bolt` — AI/power
- `bar_chart` — Metrics
- `shield` — Water resistance/protection
- `battery_charging_full` — Battery
- `directions_run` — Running/distance
- `local_fire_department` — Calories
- `water_drop` — SpO2/hydration
- `favorite` — Heart/BPM
- `check_circle` — Feature list checkmarks
- `arrow_forward` — Button arrows
- `arrow_back` — Navigation back
- `format_quote` — Testimonial quote mark
- `add` — FAQ expand (rotates 45deg to become X)
- `star` — Rating stars
- `public`, `monitoring`, `contact_support` — Footer social icons

**Active dot indicator (wide bar vs circle)**:

## Stitch Workflow Tips

1. **Create project first**: `create_project` with a descriptive name
2. **Generate screens one at a time**: Be specific in prompts — describe layout, content, and style
3. **Get screen HTML**: Use `get_screen` with `screen_id` to retrieve the download URL
4. **Download via curl**: `curl -sL <download_url> -o filename.html` (WebFetch summarizes instead of returning raw HTML)
5. **Screens may take time**: If `list_screens` returns empty, wait and retry
6. **Edit screens**: Use `edit_screens` to modify existing screens with new instructions
7. **Generate variants**: Use `generate_variants` to get alternative versions of a screen

## Integration Strategy

When merging Stitch output into an existing project:
1. Extract design tokens from Stitch's inline `tailwind.config`
2. Map Stitch Tailwind classes to your custom CSS variables
3. Replace Tailwind CDN with your own build system if needed
4. Keep Material Symbols Outlined font link — lightweight icon system
5. Preserve your existing JS functionality (scroll reveals, counters, carousels)
6. Use Stitch layouts as structural reference, adapt to your CSS methodology (BEM, etc.)
